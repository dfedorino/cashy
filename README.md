# Разработать backend-приложение для управления личными финансами

## Функциональные требования

* возможность пользователям добавлять расходы и доходы
* просматривать статистику по тратам
* устанавливать бюджеты и категории расходов

---

## Технические требования

* ~~Авторизация пользователей - реализовать функциональность для авторизации 
пользователей по логину и паролю. Приложение должно поддерживать несколько 
пользователей~~.
* Управление финансами - разработать логику для добавления доходов и расходов. 
Пользователь должен иметь возможность создавать категории для планирования бюджета. 
Установление бюджета на каждую категорию расходов.
* Кошелёк пользователя - кошелёк должен хранить информацию о текущих финансах и 
всех операциях (доходах и расходах).
* Вывод информации - отображение общей суммы доходов и расходов, а также данных по 
каждой категории. Выводить информацию о текущем состоянии бюджета для каждой 
категории, а также оставшийся лимит.
* Подсчет расходов и доходов - Разработать методы, подсчитывающие общие расходы и 
доходы, а также по категориям, подсчет по нескольким выбранным категориям, если 
категория не найдена, уведомлять пользователя.
* Проверка вводимых данных - валидация пользовательского ввода и уведомление о 
некорректных данных.
* Оповещения - оповещать пользователя, если превышен лимит бюджета по категории 
или расходы превысили доходы.
* Сохранение данных - При выходе из приложения сохранять данные кошелька 
пользователя в файл. При авторизации загружать данные кошелька из файла.
* Чтение команд пользователя в цикле - реализовать цикл для постоянного чтения 
команд пользователя. Поддержать возможность выхода из приложения.

---

## Дополнительное задание

Реализовать возможность переводов между кошельками пользователей. При переводе 
фиксировать расход у отправителя и доход у получателя, идентифицируя их по 
логину.

---

## Пример работы

Предположим, что были введены следующие данные:

Расходы:
* Еда: 300
* Еда: 500
* Развлечения: 3000
* Коммунальные услуги: 3000
* Такси: 1500

Доходы:
* Зарплата: 20000
* Зарплата: 40000
* Бонус: 3000

Бюджеты:
* Еда: 4000
* Развлечения: 3000
* Коммунальные услуги: 2500

---

После выполнения подсчетов приложение должно выводить следующую информацию:

Общий доход: 63,000.0
Доходы по категориям:
* Зарплата: 60,000.0
* Бонус: 3000.0

Общие расходы: 8,300.0
Бюджет по категориям:
* Коммунальные услуги: 2500.0, Оставшийся бюджет: -500.0
* Еда: 4000.0, Оставшийся бюджет: 3200.0
* Развлечения: 3000.0, Оставшийся бюджет: 0.0

# Критерии оценивания

## Базовый уровень сложности (основные функции)
1. Авторизация пользователей - Стабильная авторизация по логину и паролю, 
несколько пользователей
2. CLI-взаимодействие и цикл команд - Устойчивое чтение команд, корректный выход
3. Доходы и расходы - Надёжное добавление и учёт операций
4. Кошелёк пользователя - Привязан к авторизованному пользователю, хранит баланс 
и все операции
5. Категории и бюджеты - Корректные бюджеты по категориям, учитываются в расчётах
6. Вывод информации и статистика - Сводки по категориям и текущий остаток 
бюджета по каждой категории
7. Оповещения - Уведомления в нужный момент и в понятной форме
8. Сохранение и загрузка данных - При выходе пишет файл кошелька, при авторизации 
загружает корректно
9. Валидация ввода - Проверка форматов, чисел, пустых значений, неизвестных 
категорий с понятными сообщениями

## Средний уровень сложности
10. Разделение по классам - Разумные слои (модель/сервис/CLI), чёткие обязанности
11. Подсчёты по выборкам (несколько категорий) - Гибкий выбор категорий, корректные 
уведомления при отсутствии данных
12. Редактирование бюджетов и категорий - Полноценное редактирование с валидацией 
и контролем логики
13. Экспорт/импорт отчётов (CSV/JSON) - Есть экспорт и импорт данных или снимков 
состояния
14. Расширенные оповещения - Дополнительные уведомления (например, при 80% 
лимита, перерасходе, нулевом балансе)
15. UX CLI (help, подсказки, форматирование) - Команда help, примеры 
использования, форматированные таблицы и понятные ошибки

## Высокий уровень сложности
16. Тестирование - Unit- и интеграционные тесты по ТЗ, ≥ 15 тестов или 
покрытие ≥ 50%
17. Архитектура и качество кода - Чистая архитектура (core/infra/CLI), 
интерфейсы для хранилищ, читаемый код без дублирования
18. Автоматизация и сборка - Maven/Gradle + автоматическая проверка (Checkstyle, 
Spotless, GitHub Actions)
19. Документация (README + архитектура) - README с инструкцией, примерами команд, 
описанием архитектуры, указанием запуска тестов
